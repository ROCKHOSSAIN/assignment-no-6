1.Let's Discuss the scope of var , let and const

--Var--:

  • Var is a keyword of Variable
  • Var is a global variable and it can be accessible from anywhere
  • Var are functional scope and they can access within the function
  • Var are hoisted to the top of their containing function or global scope
  • Var value can be reassigned and redeclared

--Let--:

  • Let is a block scoped
  • Let can be accesible only within the block
  • Let can be hoisted but the value initialized as undefined .Because the declaration have a dead-zone where it has been hoisted
  • Let cant be redeclared
  • Let Cant be reinitialized

--Const--:

  • Const mean the value is constant that cant be changed
  • Const is a block scope that can be accessible within block
  • Const Cant be redeclared
  • Const Cant be reinitialized

2.Tell us the use cases of null and undefined

--Null--:

  • Null means there is no value and no assignned value
  • Its often used when need to make it clear that the value should be empty or undefned
  • Null can be defined in object so that it can understand that the objects should be clear and reset

--Undefined--:

  • Undefined happens when a variable has been declared but there is no value initialized
  • When we want to access object properties but that doesnt exist
  • During callback function undefined is certainly used to indicate that a function has no value or that an error occurred!!

3. What do you mean by REST API?

--Rest Api--:

  • REST API stands for Representational State Transfer Application Programming Interface.
  • REST APIs are widely used for building web services
  • While requesting information from server by the clients,there should be all the information.It should be easy to cache
  • REST APIs often use standard HTTP methods (GET, POST, PUT, DELETE) to perform CRUD (Create, Read, Update, Delete) operations on resources.
